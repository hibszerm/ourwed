# OurWed — UX Principles

Version: 1.0

---

# UX Philosophy

OurWed is not designed as a CRM.

It is designed as a daily operating system for wedding creators.

The interface should always answer the most important question:

"What do I need to do right now?"

Every screen should reduce thinking.

The application should make decisions obvious.

---

# The Five Second Rule

After opening the application, the user should know within five seconds:

What wedding is next.

How many days remain.

What requires attention.

What should be done today.

Nothing else is more important.

---

# Information Hierarchy

Priority 1

Upcoming wedding

Priority 2

Things blocking progress

Priority 3

Today's tasks

Priority 4

Upcoming deadlines

Priority 5

Everything else

---

# Navigation Philosophy

The navigation should stay minimal.

Dashboard

Weddings

Calendar

Equipment

Tasks

Settings

Nothing more.

Additional features should live inside a wedding.

Never inside the sidebar.

---

# Wedding First

The wedding is the center of the application.

Everything belongs to a wedding.

Payments

Documents

Equipment

Timeline

Questionnaires

Tasks

Notes

History

Notifications

Everything.

---

# Less Clicking

The user should never need to click through five screens to find important information.

Important actions should be visible immediately.

---

# Progressive Disclosure

Show only what matters.

Reveal details only when needed.

The interface should never overwhelm the user.

---

# Empty States

An empty screen should always explain:

what this module does

why it exists

how to start

Never show a blank page.

---

# Forms

Forms should be divided into logical sections.

Avoid long scrolling forms.

Use autosave whenever possible.

---

# Dashboard Philosophy

Dashboard is not statistics.

Dashboard is today's command center.

It answers:

What wedding is next?

What is missing?

What should I do today?

What deadline is approaching?

---

# Wedding Details Philosophy

Wedding Details is the heart of the application.

It should contain everything related to one wedding.

The user should never need to leave this screen during normal work.

---

# Calm Interface

Whitespace is a feature.

Animations should be subtle.

Typography should guide attention.

Avoid unnecessary colors.

Avoid visual clutter.

---

# Mobile Thinking

Desktop first.

Tablet second.

Mobile companion later.

Never sacrifice desktop workflow for mobile convenience.

---

# Speed

Every interaction should feel immediate.

Fast software feels premium.

---

# User Feeling

The application should make the user feel:

organized

prepared

calm

professional

in control

If the user feels stressed after opening the app, the UX has failed.
